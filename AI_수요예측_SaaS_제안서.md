## 01 현황 분석 — 생산기획의 한계

### 현재 ERP 기반 생산기획의 한계

| 문제 | 결과 | 설명 |
|------|------|------|
| 수주 급증 | 납기 지연 | 수요 변동 사전 감지 불가, 항상 사후 대응 |
| 수주 급감 | 재고 과잉 | 감산 판단이 늦어 과잉 생산·재고 비용 증가 |
| 엑셀 수작업 의존 | 속인화 리스크 | 담당자 경험·감에 의존한 생산계획 수립 |
| 외부 변수 미반영 | 예측 불가 | 환율·반도체 시장·거시경제 변동 예측 반영 불가 |

### 핵심 인사이트

> "문제는 데이터가 없는 게 아니라,
> 데이터를 미래 의사결정으로
> 연결하는 구조가 없다는 것"

- ERP는 과거 기록 시스템
- 수주 조회는 가능하지만 예측은 불가
- 재고 현황은 보이지만 리스크 감지는 불가
- 생산 실적은 있지만 최적 생산량 산출은 불가

> **본 솔루션은 ERP를 대체하는 것이 아닌, ERP 위에 얹는 AI 의사결정 레이어입니다**

| 03 | **빠른 도입** | 기존 ERP 데이터만으로 시작 가능. 3개월 내 PoC 완료 |

## 03 핵심 기능 — 5가지 핵심 모듈

### 01 AI 수요 예측

제품별 확률 밴드 예측
- P10(낙관) / P50(중앙) / P90(비관)
- 주간: 1w / 2w / 4w
- 월간: 1m / 3m / 6m
- LightGBM Quantile Regression

### 02 외부 변수 반영

30+개 경제·산업 지표 반영
- 환율(4통화), SOX지수, DRAM현물가
- FRED / EIA / ECOS / 관세청 API
- 거시경제·무역·원자재 자동 수집

### 03 리스크 스코어링

4유형 자동 평가 + A~F 등급
- 결품 리스크 (35%)
- 과잉 리스크 (25%)
- 납기 리스크 (25%)
- 마진 리스크 (15%)

### 04 생산 최적화

예측+재고+캐파 기반 최적 생산량
- 리스크별 동적 조정
- 결품 위험 시 P90 상향
- 과잉 위험 시 감량
- 긴급수주 자동 반영

### 05 발주 최적화

BOM 전개 + EOQ/ROP 기반 추천
- 공급사 가중점수 추천
- 리드타임·단가·신뢰도 종합
- 대체 공급사 자동 제시
- 발주 일정·긴급도 산출

## 06 방법론 — AI/ML 모델링 전략

### 수요예측 모델 — LightGBM Quantile Regression

#### 모델 선정 사유

- P10/P50/P90 분위 예측 직접 산출
- 외부 피처(환율, 지수) 활용 용이
- 범주형 변수 네이티브 처리
- 빠른 학습 속도, 과적합 방지

#### 피처 엔지니어링 (10개 카테고리)

| 카테고리 | 피처 내용 |
|----------|----------|
| A. 수주 이력 래그 | lag1~52주, 이동평균(MA4/MA13/MA26) |
| B. 모멘텀 | 변화율(ROC), 차분 |
| C. 변동성 | 표준편차, CV, 비영수주율 |
| D. 공급측 | 생산, 매출, 재고, Book-to-Bill 비율 |
| E. 고객 집중도 | HHI, TOP1/3 고객 비율 |
| F. 가격 | 평균단가, 주문평균값 |
| G. 반도체 시장 | SOX, DRAM, NAND, 웨이퍼 ASP |
| H. 환율 | USD/JPY/EUR/CNY → KRW |
| I. 거시경제 | 금리, 산업생산, CPI, PMI |
| J. 무역 | 반도체 HS코드 수출입, 무역수지 |
| K. 시간 | 주차, 월, 분기, 연말 여부 |

#### Fallback 전략

- LightGBM 미설치 시 이동평균 자동 전환
- 학습 데이터 부족 제품: 가중 이동평균 적용

### 리스크 스코어링 + 생산·발주 최적화

#### 리스크 스코어링 (S5)

| 리스크 유형 | 가중치 | 산출 기준 |
|-------------|--------|----------|
| 결품 리스크 | 35% | P90수요 대비 재고 부족도 |
| 과잉 리스크 | 25% | 재고일수 90일 초과 기준 |
| 납기 리스크 | 25% | P90 리드타임 대비 잔여일 |
| 마진 리스크 | 15% | 원가 상승·환율 변동 압박 |

등급 체계: **A**(0~20) / **B**(21~40) / **C**(41~60) / **D**(61~80) / **F**(81~100)

#### 생산 최적화 (S7)

- 수요예측 + 현재재고 + 안전재고 → 순소요량
- 생산 캐파시티 제약 반영 (x1.2 버퍼)
- 리스크 등급별 동적 조정
  - D/F + 결품 위험 ↑ → P90 기반 증산
  - D/F + 과잉 위험 ↑ → 10% 감량
- 미처리 긴급수주 자동 반영

#### 발주 최적화 (S8)

- BOM 전개 → 자재별 총소요량 산출
- 안전재고·재주문점(ROP) 산출
- EOQ + Lot-for-Lot 하이브리드
- 공급사 가중점수 추천
  - 리드타임 40% + 단가 35% + 신뢰도 25%
- 대체 공급사 자동 제시

## 07+ 검증 방법론 — 신뢰성 확보 전략

### 1단계: 모델 검증 — Walk-Forward Cross Validation

- 시계열 특성을 반영한 전진 교차검증
- 주간 3-fold / 월간 2-fold
- Pinball Loss 기반 분위 예측 정확도
- Grid Search 하이퍼파라미터 튜닝
- 예측 vs 실적 비교 리포트

### 2단계: 로직 검증 — 리스크 스코어링 정합성

- 등급 경계 연속성 검증 (갭 버그 수정)
- 가중치 민감도 분석 (3개 시나리오)
  - 모든 시나리오에서 C이상 비율 11~16% 안정
- 수정 전/후 비교: critical 66% → 3%
- 조치 큐 심각도 분포 정상화 확인

### 3단계: 결과 검증 — 생산·발주 최적화 무결성

- 음수 값 발생 여부 (0건 확인)
- 일정 정합성 (발주일 <= 필요일)
- JSON 포맷 유효성
- 우선순위·긴급도 분포 합리성
- 공급사 매핑율 확인
- planned_qty=0 & critical 이상치 추적

> **지속적 검증**: 예측 vs 실적 자동 비교 리포트 | 주간 모델 재학습 | 리스크 스코어 추이 모니터링 | A/B 테스트
