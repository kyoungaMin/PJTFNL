## 3. 차트 영역 (Row 2) 상세 검토

### 3-1. 매출 실적 추이 차트

| 항목 | 내용 |
|------|------|
| **현재 상태** | 실적만 DB 연동 (`daily_revenue`), P10/P50/P90 예측 밴드는 Mock |
| **DB 연동 가능 여부** | **△ (부분 가능)** |
| **관련 테이블** | `forecast_result` — 수주량(order_qty) 예측값 P10/P50/P90 존재. 단, 매출 예측이 아닌 **수주 예측** |
| **사용자 유의미성** | **△ (변경 검토 필요)** |
| **사유** | 매출(revenue)은 생산계획팀에게 **후행 지표**. 수주량(order)은 생산 투입의 직접적 선행 지표이므로 **수주량 실적 + AI 예측 추이** 차트가 더 유의미 |
| **권고** | "매출 실적 추이" → **"수주량 추이 + AI 예측"** 으로 변경 검토 |

**변경 시 데이터 소스:**
- 수주 실적: `daily_order` → 월별 집계 또는 `monthly_product_summary.order_qty`
- AI 예측: `forecast_result` → P10/P50/P90 밴드

### 3-2. 위험 품목 현황 (도넛 차트)

| 항목 | 내용 |
|------|------|
| **현재 상태** | 완전 Mock (A:187, B:143, C:89, D:45, E:28, F:12 하드코딩) |
| **DB 연동 가능 여부** | **O** — `risk_score` 테이블에 `risk_grade` 컬럼 (A~F) 존재 |
| **사용자 유의미성** | **O (유의미)** |
| **사유** | 위험 등급 분포를 한눈에 파악하여 전체 리스크 현황을 모니터링. E~F 등급 클릭 시 리스크 관리 페이지로 이동하는 UX도 적절 |
| **권고** | `risk_score` 테이블에서 등급별 COUNT 집계로 DB 연동 |

**연동 쿼리 예시:**
```sql
SELECT risk_grade, COUNT(*) as count
FROM risk_score
WHERE eval_date = (SELECT MAX(eval_date) FROM risk_score)
GROUP BY risk_grade
ORDER BY risk_grade;
```

## 6. 대시보드에 추가 검토 가능한 지표

현재 DB에 데이터가 존재하지만 대시보드에 표시되지 않는 유의미 지표:

| 지표 | 데이터 소스 | 유의미도 | 사유 |
|------|------------|---------|------|
| **모델 정확도 (MAPE)** | `model_evaluation` 테이블 | 높음 | AI 수요예측 모델의 신뢰도를 한눈에 확인. 예측 기반 의사결정의 근거 |
| **금주 생산 계획 달성률** | `production_plan` + `daily_production` | 높음 | 계획 대비 실적 갭을 모니터링하여 생산 일정 지연 조기 감지 |
| **환율 동향 (USD/KRW)** | `exchange_rate` 테이블 | 중간 | 수입 부품 원가에 직접 영향. 급등 시 구매 시점 조정 필요 |
| **주간 수주 추이** | `weekly_product_summary` | 중간 | 최근 수주 트렌드로 수요 변동 감지 |

## 7. 우선순위 기반 개선 로드맵 (제안)

### Phase 1 — 즉시 적용 (Mock → DB 전환)
1. AI 생산 권고 KPI → `production_plan` 연동
2. 긴급 대응 SKU KPI → `risk_score` 연동
3. 위험 품목 도넛 차트 → `risk_score` 연동
4. AI 생산 권고사항 → `action_queue` 연동
5. 구매 발주 KPI → `purchase_recommendation` 긴급 건수로 변경

### Phase 2 — 차트 개선
6. 매출 실적 추이 → 수주량 추이 + AI 예측 밴드로 변경
7. 모델 정확도(MAPE) KPI 또는 뱃지 추가

### Phase 3 — 인사이트 고도화
8. AI 인사이트 → 데이터 기반 자동 요약 또는 제거
